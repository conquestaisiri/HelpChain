✅ WHAT SHOULD SHOW WHEN YOU CLICK “OFFER HELP”
🔹 CASE 1: USER IS NOT LOGGED IN

If a logged-out user taps Offer Help:

What should happen:

A modal pops up saying:

“You must log in to offer help.”

With two buttons:

Login

Create Account

After clicking, it redirects to the login/signup page.

NOTHING ELSE should show.

NO form.
NO additional screen.
Just login requirement.

🔹 CASE 2: USER IS LOGGED IN

If the user is already signed in, clicking Offer Help should open a pop-up modal or a full page titled:

“Offer Your Help”

Inside this, the following must appear:

✅ THE OFFER HELP FORM (FULL DETAILS)
1. Helper Profile Preview (Top Section)

Shows:

Helper’s name

Profile picture

“You’re offering help to [Requester’s Name]”

Just context.

2. Message Box (Main Component)

A typing area labeled:

“Send a message introducing yourself and explaining how you can help.”

Default placeholder text:

“Hi, I saw your request and I’d like to help. I have experience with this kind of work…”

This message will be visible only to the requester.

3. Availability Section

Small dropdown:

When can you start?

“Right now”

“In 30 minutes”

“In 1 hour”

“Anytime today”

“Schedule a time”

Optional but improves clarity.

4. Distance Indicator (If location is enabled)

“This request is X km near you.”

(Not required but improves trust.)

5. Confirm Button

Big button:

Submit Offer

When submitted:

The requester gets a notification

The helper is added to the list of potential helpers

The helper sees:
“Offer sent! Waiting for requester to review.”

6. Cancel Button

“Cancel”

Closes the form.

🛑 VERY IMPORTANT RULE

Submitting “Offer Help” does NOT lock anything, start a transaction, or hold money.

It is ONLY the expression of interest.

The REQUESTER must later select a helper before the transaction starts.

🔥 EXTRA DETAILS TO MAKE THE FEATURE SOLID
After the helper submits the offer:

They should see:

“Offer Pending”

“Waiting for requester selection”

They cannot send more than one offer

They can withdraw their offer anytime

Requester side:

They see:

List of helpers who offered help

Each helper’s public profile preview

Accept / Decline button for each
